## Hint เรื่องการทำ Bellman Algorithm

การทำ Bellman ปกติจะให้ตัวเริ่มเป็น 0 และที่เหลือเป็น INF เพราะการใช้ 0 พอไปบวก weight มันจะได้ค่าเท่าเดิม แต่การทำ Bellman แบบกำหนดจุดเริ่มแบบนี้ใช้ได้เฉพาะอันที่เป็น connected graph เพราะถ้าสมมติ graph มี negative cycle นอกเหนือจาก region ที่เริ่ม จะ detect ไม่เจอ เพราะจะวนอยู่แค่ที่เริ่ม เพราะงั้นการทำคือ จะสร้าง Super Source มา 1 จุด แล้วเชื่อมเข้าหาทุก node ด้วย weight = 0 หรือ อีกวิธีคือ เริ่มทุก node = 0 ไปเลย (Logic เดียวกัน แต่แค่ย่อ โดยอันนี้จะใช้เฉพาะต้องการ Detect ว่ามี Negative Cycle ไหม) เหมือนข้อ Shortest Start ที่สร้าง Super Source มาโยงถึงตัวที่ start

ในข้อนี้เป็นการประยุกต์ Bellman คือทำแบบเทียบหา cycle ที่ทำให้ผลคูณมันมีค่ามากขึ้น (ไปเรื่อยๆ จะเป็น INF) โดยเราจะให้ จุดเริ่มเป็น 1.0 และที่เหลือเป็นค่าน้อยๆ หรือ 0 (ปล.ที่ให้เริ่มเป็น 1.0 เพราะมันคูณกับอะไรก็ได้ค่าเดิม logic เดียวกับการบวกที่ให้เริ่มเป็น 0) แล้วก็ทำ Bellman ปกติเลย แต่เทียบ > แทนที่จะเป็น < แค่นั้น ที่สำคัญคือถ้าโจทย์อยากได้ Cycle ก็ทำเหมือนเดิมคือ โจทย์ไม่ได้กำหนดมาว่ามันต้องเป็น Connected Graph ดังนั้นจึงเริ่มด้วยให้ทุก node เริ่มจาก 1.0 เลย (ใช้สำหรับ Detect Cycle ที่เป็น INF)
